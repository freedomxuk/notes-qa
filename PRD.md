# Notes QA - 产品需求文档 (PRD)

## 1. 产品概述

**产品名称**: Notes QA
**产品类型**: AI 问答助手（RAG 应用）
**核心价值**: 让用户基于自己的笔记库进行问答，每个答案 claim 都可追溯到原文出处
**目标用户**: 有大量个人笔记（会议记录、研究笔记、日记、读书笔记）需要 AI 检索的人

---

## 2. 用户故事

### 作为用户，我希望...
1. 上传我的笔记文件（Markdown、TXT），让 AI 学习笔记内容
2. 用自然语言提问关于笔记内容的问题
3. 得到准确答案，并且知道答案是哪段笔记支持的
4. 点击答案中的引用，直接跳转到原文位置验证

### 边界场景
- 提问内容笔记中没有答案 → 明确告知"未找到相关信息"
- 上传格式不支持 → 提示支持的格式
- 笔记很长 → 确保超长上下文也能处理

## 3. 功能需求

### 3.1 笔记管理
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 笔记上传 | 支持 .md、.txt 文件拖拽上传，多次上传累积 | P0 |
| 笔记展示 | 左侧栏列表展示已上传笔记（文件名、字数、上传时间） | P1 |
| 笔记解析 | 解析文件内容为纯文本，按段落分块存储 | P0 |
| 预置示例笔记 | 混合内容示例笔记（产品介绍、日记、研究笔记） | P1 |

### 3.2 问答功能
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 自然语言提问 | 用户输入问题，AI 基于笔记回答 | P0 |
| 引用追溯 | 每个答案 claim 标注来源 passage，角标数字[1][2]样式 | P0 |
| 引用跳转 | 点击引用跳转到原文，右侧固定显示原文 passage | P0 |
| 引用预览 | 鼠标悬停显示气泡预览原文片段 | P1 |
| 无答案处理 | 笔记库无相关信息时，诚实告知用户 | P0 |
| 示例问题按钮 | 首页预设 3-5 个示例问题按钮 | P1 |
| 流式输出 | 答案边生成边显示 | P0 |

### 3.3 UI 交互
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 两栏布局 | 左侧笔记列表 / 右侧问答区 | P0 |
| 右侧原文栏 | 固定显示当前引用的原文 passage | P0 |

### 3.4 UI 详细设计

#### 整体风格
- **视觉风格**: 简洁白/蓝，专业干净
- **字体**: 系统默认字体（-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif）
- **主色调**: #2563EB（蓝色）
- **响应式断点**: 768px（专注桌面）

#### 左侧栏（笔记列表）
- **列表样式**: 卡片式显示（文件名 + 预览文字）
- **笔记数量**: 显示 badge（如 "12 篇笔记"）
- **上传按钮**: 顶部蓝色大按钮 "+ 添加笔记"
- **空状态**: 大引导 + 上传图标 + "上传你的第一篇笔记"

#### 右侧栏（问答区）
- **布局**: 上下结构，上半答案，下半输入
- **输入框**: 底部固定，placeholder 打字机动画
- **发送按钮**: 蓝色圆形按钮（➤ 图标）
- **示例问题**: 3-5 个预设按钮

#### 答案展示
- **文字样式**: 普通段落文字
- **引用标记**: 角标数字 [1][2]，蓝色可点击
- **悬停预览**: 鼠标悬停显示气泡原文
- **右侧原文**: 固定显示当前点击的 passage，背景高亮

#### Loading 状态
- **上传中**: 蓝色进度条 + 百分比
- **AI 生成**: 流式打字效果
- **错误提示**: 红色提示文字

## 4. 非功能需求

### 性能
- 单次响应时间 < 10秒（正常网络情况下）
- 支持最多 ~200 篇笔记（每篇几千字）

### 准确性
- 引用必须准确指向源文 passage，不能是"附近段落"
- 无答案时不能乱编，必须诚实告知

### 成本控制
- 使用 MiniMax M3（1M token 上下文）降低 LLM 成本
- 使用 OpenAI Embedding（效果好）确保语义正确性
- 预估 API 花费 < $20

## 5. 技术方案

### 架构图
```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   前端      │────▶│   API       │────▶│   存储      │
│  (Next.js)  │◀────│  (Server)   │◀────│(Vercel Blob)│
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                    ┌──────┴──────┐
                    ▼             ▼
              ┌──────────┐   ┌──────────┐
              │ Embedding│   │   M3    │
              │  OpenAI │   │ MiniMax │
              └──────────┘   └──────────┘
```

### 数据流
1. **上传流程**: 文件 → 解析文本 → 按段落分块 → Embedding → 存储在内存
2. **问答流���**: 问题 → 检索 Top-K chunks → M3 生成 → 带引用返回

### 存储设计
- **文件存储**: Vercel Blob（原始笔记文件）- **元数据存储**: 内存检索（每次启动重新处理 blob）
- **分块结构**:
  ```json
  {
    "id": "uuid",
    "source_file": "note.md",
    "chunk_index": 0,
    "content": "原文 passage...",
    "embedding": [0.1, 0.2, ...],
    "created_at": "2024-01-01"
  }
  ```

### 分块策略
- 按 Markdown 段落自然分割
- 合并过短段落（小段落合并到相邻段落）
- 每个 chunk ~200-500 tokens

## 6. 错误处理

| 场景 | 处理方案 |
|------|---------|
| embedding 接口失败 | 重试 2 次，失败提示"解析失败，请重试" |
| 问答接口超时 (>30s) | 返回"请求超时，请精简问题或减少笔记量" |
| 余额不足 | 提示检查 API 配置 |
| 没上传笔记就提问 | "请先上传笔记后再提问哦～" |
| 笔记为空搜不到 | "未找到相关笔记，请尝试其他问题或补充笔记" |
| 上传格式不支持 | 提示仅支持 .md 和 .txt |
